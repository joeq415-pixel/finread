# FinRead Beta Testing Guide

Thank you for joining our beta! Your feedback is critical to making FinRead the best financial analysis platform. This guide outlines key tasks to complete and things to pay attention to.

**Estimated Time:** 20-30 minutes  
**Goal:** Test core features and provide feedback on usability

---

## 🎯 Core Testing Tasks

### Task 1: Account Setup & Onboarding
**Objective:** Verify account creation and onboarding flow is intuitive

**Steps:**
1. Go to finread.io
2. Click "Sign Up"
3. Enter email and create password
4. Submit and check email for verification link
5. Click verification link and complete profile
6. Return to app after verification

**Things to Note:**
- [ ] Was the sign-up process clear?
- [ ] Did you receive the verification email quickly?
- [ ] Was there any confusion in the onboarding flow?
- [ ] Any errors or technical issues?

**Report Issues:** Note any error messages or confusing steps

---

### Task 2: Search & Analyze Your First Company
**Objective:** Test the core analysis feature

**Steps:**
1. Search for a company (try: Apple, Microsoft, Tesla, or your favorite)
2. Select a 10-K filing (annual report)
3. Read the AI-generated summary
4. Scroll through the detailed analysis
5. Note the key metrics displayed

**Things to Note:**
- [ ] Was it easy to find the company you wanted?
- [ ] Did the AI analysis make sense?
- [ ] Were the explanations clear and helpful?
- [ ] Did you learn something new about the company?
- [ ] Was the layout easy to read?

**Report Issues:** Unclear explanations, missing data, UI problems

---

### Task 3: Interact with AI - Ask Questions
**Objective:** Test the AI Q&A feature

**Steps:**
1. On the filing analysis page, scroll to "Ask AI Questions" section
2. Ask a specific question (e.g., "What was the company's revenue growth?")
3. Read the AI response
4. Ask 1-2 more questions about different aspects

**Things to Note:**
- [ ] Was the question input clear and easy to use?
- [ ] Were the AI responses accurate and helpful?
- [ ] Did it answer follow-up questions well?
- [ ] Response time - was it fast enough?
- [ ] Any hallucinations or incorrect information?

**Report Issues:** Inaccurate answers, slow response time, unclear interface

---

### Task 4: Explore & Add to Watchlist
**Objective:** Test watchlist feature

**Steps:**
1. Click the "Add to Watchlist" button
2. Confirm the company was added
3. Navigate to your Watchlist
4. Verify the company appears there
5. (Optional) Add 1-2 more companies

**Things to Note:**
- [ ] Was it clear how to add to watchlist?
- [ ] Did it confirm the action?
- [ ] Was accessing the watchlist intuitive?
- [ ] Is the watchlist useful for your needs?

**Report Issues:** Button not working, companies not saving, UX issues

---

### Task 5: Test Pro Features & Upgrade
**Objective:** Verify Pro upgrade flow and pricing

**Steps:**
1. Look for Pro-only features (marked with 🔒 lock icon)
2. Click "Upgrade to Pro" button
3. Review the pricing ($4.99/month)
4. **DO NOT complete payment** (test only)
5. Note the checkout flow and design
6. Click back to return to app

**Things to Note:**
- [ ] Was it clear what Pro features include?
- [ ] Was the pricing clearly displayed?
- [ ] Did the upgrade button work?
- [ ] Was the checkout process smooth?
- [ ] Any confusing steps in payment flow?
- [ ] What's your gut reaction to the price?

**Report Issues:** Broken links, unclear pricing, payment errors

---

### Task 6: Test Multiple Filing Types
**Objective:** Verify analysis works for different document types

**Steps:**
1. Search the same company again
2. Find and analyze a **10-Q** (quarterly report)
3. Compare it to the 10-K you analyzed before
4. Try finding an **8-K** (current report) if available
5. Note any differences in analysis quality

**Things to Note:**
- [ ] Was the 10-Q analysis as good as the 10-K?
- [ ] Could you easily switch between filing types?
- [ ] Any errors or missing data for certain types?
- [ ] Were quarterly insights useful?

**Report Issues:** Analysis quality drops for certain filings, missing data

---

### Task 7: Test Mobile & Responsive Design
**Objective:** Verify experience on mobile devices

**Steps:**
1. Open finread.io on your phone/tablet
2. Sign in to your account
3. Search for a company and view analysis
4. Try asking an AI question
5. Navigate between pages
6. Attempt to upgrade to Pro

**Things to Note:**
- [ ] Is the layout readable on mobile?
- [ ] Are buttons easy to tap?
- [ ] Is scrolling smooth?
- [ ] Do all features work on mobile?
- [ ] Any broken layouts or overlapping text?
- [ ] Loading times acceptable?

**Report Issues:** Mobile layout broken, buttons unresponsive, text too small

---

### Task 8: Test Light & Dark Mode
**Objective:** Verify visual design in both themes

**Steps:**
1. Toggle between Light and Dark modes (usually in settings)
2. View the same filing in both modes
3. Pay attention to readability and contrast
4. Check if all elements are visible in both modes

**Things to Note:**
- [ ] Is light mode readable and visually appealing?
- [ ] Is dark mode easy on the eyes?
- [ ] Are all buttons and text visible in both modes?
- [ ] Any colors that don't work well together?
- [ ] Which mode do you prefer?

**Report Issues:** Poor contrast, invisible text, broken themes

---

### Task 9: Test Error Handling
**Objective:** See how the app handles problems

**Steps:**
1. Try searching for a company that doesn't exist (e.g., "XYZ Corp Fake")
2. Try going back in your browser history repeatedly
3. Disconnect from internet briefly (if possible)
4. Try searching with special characters or numbers
5. Try uploading/exporting if features exist

**Things to Note:**
- [ ] Did you get helpful error messages?
- [ ] Did the app recover gracefully?
- [ ] Were error messages clear about what went wrong?
- [ ] Any crashes or frozen screens?

**Report Issues:** Unclear errors, crashes, unhelpful messages

---

### Task 10: Performance & Speed Testing
**Objective:** Evaluate performance

**Steps:**
1. Time how long it takes to search for a company (target: < 2 seconds)
2. Time how long AI analysis takes to load (target: < 5 seconds)
3. Note if there's any lag when scrolling
4. Check if images/charts load properly
5. Try rapid clicking/navigation

**Things to Note:**
- [ ] Is the app fast or slow?
- [ ] Any lag or stuttering?
- [ ] Do charts/images load clearly?
- [ ] Does it handle rapid interactions well?

**Report Issues:** Slow performance, lag, timeouts

## 🐛 How to Report Issues

**Found a bug?** Please include:

1. **What you were doing** (step-by-step)
2. **What happened** (actual result)
3. **What should have happened** (expected result)
4. **Screenshot** (if possible)
5. **Device/Browser** (e.g., iPhone 14 Safari, MacBook Chrome)
6. **When it happened** (date/time)

**Example Bug Report:**
> "When I searched for 'Apple Inc' on mobile, the results showed 3 different companies all named Apple. I clicked the second one, but it took me to a different company's page. Expected: Correct company loaded."
